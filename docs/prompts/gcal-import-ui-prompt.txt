OBJECTIVE
---------
Build a "Import from Google Calendar" feature into the existing Booking admin
module. The coach can preview all existing calendar events, individually
approve or skip each one, then import only the approved ones into MongoDB.

This replaces the CLI script approach. No standalone script is needed.
All logic lives in the Angular booking module + existing Node.js API.

Check CLAUDE.md before writing any code.

================================================================================

================================================================================

  Existing: src/app/features/booking/admin/booking-dashboard/
  Add tab:  "Import from Calendar" (only shown if importedCount === 0,
             OR always accessible via an "Import more" button after first import)

BACKEND — New API Endpoints
================================================================================

TASK B.1 — Preview endpoint

  GET /api/booking/import/preview

  Auth: JWT required (existing middleware)
  Query params:
    from    ISO date string, default: 2 years ago
    to      ISO date string, default: 1 year from now

  Steps:
    1. Load AvailabilityConfig for the authenticated coach → get targetCalendarId
    2. Call getOAuth2Client(coachId) — reuse existing OAuth service
    3. Fetch all events from Google Calendar via paginated events.list():
         calendarId:   targetCalendarId
         timeMin:      from
         timeMax:      to
         singleEvents: true       ← expand recurring events
         showDeleted:  false
         orderBy:      'startTime'
         maxResults:   250 per page
    4. Filter events (see Filtering Rules below)
    5. For each filtered event, check if a Booking already exists in MongoDB
       with matching googleEventId + coachId → set alreadyImported: true
    6. Return the event list with import status

  Response shape:
    {
      total:           number,    // total fetched from GCal
      filtered:        number,    // after applying filters
      alreadyImported: number,    // subset already in our DB
      events: [
        {
          googleEventId:   string,
          title:           string,    // event.summary
          startTime:       ISO string,
          endTime:         ISO string,
          durationMinutes: number,
          clientName:      string,    // derived from attendees
          clientEmail:     string,    // derived from attendees
          clientPhone:     null,
          topic:           string,    // event.description
          googleMeetLink:  string | null,
          status:          'upcoming' | 'completed',  // based on startTime vs now
          alreadyImported: boolean,
          rawSummary:      string,    // original event.summary for display
          attendees:       [{ email, displayName, self, organizer }]
        }
      ]
    }

  Filtering Rules (same as CLI script — apply server-side):
    - Skip all-day events (no event.start.dateTime)
    - Skip events shorter than 15 minutes
    - Skip events where attendees count <= 1 (solo personal blocks)
    - Skip events with status !== 'confirmed'

  Do NOT filter out alreadyImported events — return them so the UI can
  show them as greyed out / skipped.

  Performance note: if the calendar has 500+ events, this may take a few
  seconds. The frontend must show a loading spinner with a progress message.

TASK B.2 — Import endpoint

  POST /api/booking/import/execute

  Auth: JWT required
  Body:
    {
      approvedEventIds: [string]   // array of googleEventIds the coach approved
    }

  Steps:
    1. Load AvailabilityConfig (for coachId, tenantId, timezone)
    2. For each approvedEventId:
       a. Fetch the event from Google Calendar API to get current data
          (do not trust the preview payload — re-fetch for accuracy)
       b. Check Booking collection: if googleEventId already exists → skip (idempotent)
       c. Map event to Booking document (see Mapping Rules below)
       d. Insert new Booking with importedAt and importSource fields
    3. Return:
       {
         imported: number,
         skipped:  number,    // already existed
         errors:   [{ googleEventId, message }]
       }

  Mapping Rules:
    clientName:     first non-coach attendee displayName, or derived from email,
                    or event.summary if no attendees
    clientEmail:    first non-coach attendee email, or null
    topic:          event.description or null
    startTime:      new Date(event.start.dateTime)
    endTime:        new Date(event.end.dateTime)
    clientTimezone: event.start.timeZone or null
    coachTimezone:  config.timezone
    googleEventId:  event.id
    googleMeetLink: event.hangoutLink or conferenceData video entry URI or null
    cancelToken:    null  (no cancel-by-link for imported events)
    status:         'completed' if endTime < now, else 'confirmed'
    remindersSent:  []
    importedAt:     new Date()
    importSource:   'gcal_import_ui'

  Add importedAt and importSource to Booking schema if not already present:
    importedAt:   { type: Date, default: null }
    importSource: { type: String, default: null }

================================================================================
EDGE CASES TO HANDLE
================================================================================

  E1 — Coach not connected to Google Calendar
       If getOAuth2Client returns null, the preview endpoint returns 401.
       Frontend: detect this error and show:
         "Connect your Google Calendar first (Settings tab) before importing."

  E2 — Calendar has 0 events in range
       Return empty events array. Frontend shows:
         "No coaching sessions found in this date range."
       with suggestion to widen the date range.

  E3 — Preview returns events but coach approves 0
       "Import X Events" button stays disabled. No API call made.

  E4 — Import partially fails (some events error)
       Show DONE state with errors section. Do not block the successful imports.

  E5 — Coach runs import twice (second run)
       All events will have alreadyImported: true.
       Preview shows: "0 pending — all sessions already imported."
       Import button disabled.

  E6 — Event has no attendee email (just Helena in the event)
       clientEmail: null, clientName: derived from event title.
       Show a warning icon on that row in the preview table:
         "⚠ No client email found — session will import without email."

  E7 — Very large calendar (200+ events)
       Preview endpoint may take 5–10 seconds due to pagination.
       Show "Fetching page X..." updates using SSE or just a static spinner.
       Set axios/http timeout to 30 seconds for this endpoint.
