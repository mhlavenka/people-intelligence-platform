CONTEXT
-------
You are implementing a Calendly-equivalent appointment booking module for the
Artes application (People Intelligence Platform). The target user is Helena, a
PCC-certified executive coach. The module allows coaching clients to self-book
sessions via a public link, with appointments written to a specific Google
Calendar ("Coaching") — not her primary calendar.

This is a lazy-loaded Angular module integrated into the existing Artes codebase.
Do not create a standalone app.

TECH STACK
----------
- Frontend:  Angular (latest in project), Angular Material, Luxon for dates
- Backend:   Node.js / Express, existing middleware and auth patterns
- Database:  MongoDB / Mongoose, scoped by coachId and tenantId
- Auth:      Existing JWT middleware — reuse as-is for admin endpoints
- Email:     AWS SES (existing SES setup in project)
- External:  Google Calendar API v3 via googleapis npm package
- Scheduler: node-cron for reminder jobs

EXISTING CONVENTIONS TO FOLLOW
-------------------------------
- Check CLAUDE.md for project-specific instructions before writing any code
- Match existing folder structure, naming conventions, and code style
- Reuse existing: AuthGuard, TenantGuard, ApiService base class, error handling interceptor
- All Mongoose models must include tenantId: { type: ObjectId, ref: 'Tenant', required: true }
- Use environment.ts / environment.prod.ts for all config — no hardcoded values

  Store as: iv:authTag:ciphertext (colon-separated hex strings)

TASK 1.3 — Create Google OAuth service
  File: src/services/google-oauth.service.js

  Methods:
    getAuthUrl(coachId)
      - Returns Google OAuth2 authorization URL
      - Scopes: calendar.readonly, calendar.events, calendar.calendarlist.readonly
      - access_type: 'offline', prompt: 'consent'
      - State param: sign JWT containing coachId (verify on callback)

    handleCallback(code, state)
      - Verify state JWT
      - Exchange code for tokens via oauth2Client.getToken()
      - Encrypt and save to OAuthToken collection
      - Fetch and cache calendar list
      - Return { success: true, coachId }

    getOAuth2Client(coachId)
      - Load OAuthToken for coachId
      - Decrypt tokens
      - Instantiate OAuth2Client with credentials
      - If token expires within 5 minutes, call refreshTokens() first
      - On 401 from any Google API call: attempt one refresh, then retry
      - If refresh fails: mark coach disconnected (set token to null), throw error

    refreshTokens(coachId)
      - Use refresh_token to get new access_token
      - Update OAuthToken with new accessToken and expiresAt

    revokeAndDisconnect(coachId)
      - Call oauth2Client.revokeToken()
      - Delete OAuthToken document for coachId

    listCalendars(coachId)
      - Use authenticated client to call calendarList.list()
      - Cache result in OAuthToken.calendarList
      - Return array of { id, summary, primary }

PHASE 3: BACKEND — Availability Engine
================================================================================

TASK 3.1 — Create availability service
  File: src/services/availability.service.js

  Use luxon (DateTime) for all date/time operations. Never use moment.js.
  Use node-cache for caching with 2-minute TTL.

  Method: getAvailableSlots(coachSlug, fromDate, toDate, clientTimezone)

  Algorithm:
    1. Load AvailabilityConfig by coachSlug
    2. Validate date range: max 60 days, from >= today + minNoticeHours
    3. For each day in range:
       a. Get weeklySchedule entry for that dayOfWeek (skip if !enabled)
       b. Check dateOverrides: if isUnavailable → skip day entirely
                               if override exists → use override times instead
       c. Generate slots: from startTime to endTime, step = appointmentDuration + bufferTime
       d. Enforce minNoticeHours: skip slots where startTime < now + minNoticeHours
       e. Track bookingsThisDay; stop generating if maxBookingsPerDay reached
    4. Collect all candidate slot start/end times (store as UTC)
    5. Fetch busy periods from Google Calendar API (freebusy query):
       - calendars: conflictCalendarIds + targetCalendarId
       - timeMin: start of date range (UTC ISO)
       - timeMax: end of date range (UTC ISO)
    6. Also query MongoDB Booking collection for confirmed bookings in range
       (handles case where Google event creation failed but booking was saved)
    7. Remove any candidate slot that overlaps any busy period
    8. Return slots as array of { startUtc, endUtc, startLocal, endLocal, label }
       where local = converted to clientTimezone

  Cache key: `slots:${coachSlug}:${fromDate}:${toDate}`
  Invalidate cache: on config update, on new booking, on webhook received

TASK 3.2 — Cache invalidation helper
  File: src/services/cache.service.js
  Simple wrapper around NodeCache with invalidateByPrefix(prefix) method

PHASE 4: BACKEND — Booking Service
================================================================================

TASK 4.1 — Create booking service
  File: src/services/booking.service.js

  Method: createBooking(coachSlug, bookingData)
    1. Load AvailabilityConfig
    2. Re-validate the requested slot is still available (race condition protection)
       - Check Google Calendar freebusy for that exact slot
       - Check Booking collection for any confirmed booking overlapping that slot
       - If slot taken: throw SlotUnavailableError
    3. Create Google Calendar event:
       - calendar: targetCalendarId
       - summary: `Coaching Session — ${clientName}`
       - description: topic (if provided)
       - start/end: slot times in coach timezone
       - attendees: [{ email: clientEmail }, { email: coachEmail }]
       - conferenceData: if googleMeetEnabled, add createRequest
       - sendUpdates: 'none'  (we send our own emails)
    4. Save Booking to MongoDB with googleEventId and googleMeetLink
    5. Generate cancelToken: sign JWT { bookingId, type: 'cancel' }, 7-day expiry
    6. Save cancelToken to Booking document
    7. Send confirmation emails (coach + client) — see Notification service
    8. Invalidate availability cache for coachSlug
    9. Return booking document

  Method: cancelBooking(bookingId, cancelledBy, reason)
    1. Load booking, verify status === 'confirmed'
    2. Update Google Calendar event: set status to 'cancelled' or delete event
    3. Update Booking: status → 'cancelled', cancelledAt, cancellationReason
    4. Send cancellation email to the other party
    5. Invalidate cache

TASK 5.2 — Create reminder scheduler
  File: src/jobs/reminder.job.js

  Run every 15 minutes via node-cron: '*/15 * * * *'

  Logic:
    const now = new Date();
    const in24h = new Date(now.getTime() + 24 * 60 * 60 * 1000);
    const in1h  = new Date(now.getTime() + 60 * 60 * 1000);
    const window = 30 * 60 * 1000;  // 30-minute window

    Query Booking collection:
      status: 'confirmed'
      startTime: { $gte: now }

    For each booking:
      - If startTime within [in24h - window, in24h + window] AND '24h' not in remindersSent:
          sendReminder(booking, '24h')
          push { type: '24h', sentAt: now } to remindersSent
      - If startTime within [in1h - window, in1h + window] AND '1h' not in remindersSent:
          sendReminder(booking, '1h')
          push { type: '1h', sentAt: now } to remindersSent

  Register job in main app entry point (app.js or server.js)
