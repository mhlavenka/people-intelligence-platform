Helena Coaching has approximately 100 historical MHS EQi 2.0 PDF assessment

  Mode A — IDENTIFIED
    Full client profile stored encrypted. Name, email, role preserved.

MHS EQi 2.0 REPORT STRUCTURE — WHAT TO PARSE
================================================================================

MHS produces several report types. This importer must handle all of them.
The score layout differs slightly between types but the 15 subscales and
5 composites are always present.

---- REPORT TYPES TO SUPPORT ----

  1. EQi 2.0 Workplace Report      (most common — ~80% of Helena's reports)
  2. EQi 2.0 Leadership Report     (second most common)
  3. EQi 2.0 Group Report          (aggregated — handle separately, see below)
  4. EQ 360 Workplace Report       (multi-rater — self + observer scores)
  5. EQ 360 Leadership Report      (multi-rater leadership context)

---- THE 20 SCORES TO EXTRACT FROM EVERY INDIVIDUAL REPORT ----

The 1-5-15 model: 1 Total EI / 5 Composites / 15 Subscales.
All scores are standard scores (mean=100, SD=15). Valid range: 55–145.

  WELL-BEING INDICATOR             (separate — not part of Total EI)

---- METADATA TO EXTRACT WHERE PRESENT ----

  Report type (Workplace / Leadership / Group / 360)

  Norm group (e.g. General Population, Professional Global)

================================================================================
PDF PARSING STRATEGY
================================================================================

Use the npm package 'pdf-parse' for text extraction.
Install: npm install pdf-parse @types/pdf-parse --save

MHS report PDFs contain machine-readable text (not scanned images), so
text extraction works reliably. However, the layout varies between report
versions and types, so the parser must be robust to variation.

---- PARSING APPROACH ----

Step 1: Extract full text from PDF using pdf-parse
Step 2: Run score extraction using pattern matching (see patterns below)
Step 3: Validate all extracted scores (range 55-145, all 20 fields present)
Step 4: If any score fails validation, flag for manual review — do NOT import
Step 5: Extract metadata fields using secondary patterns
Step 6: Return structured ParseResult object (see schema below)

---- SCORE EXTRACTION PATTERNS ----

MHS reports render scores in several formats across versions.
The parser must try ALL patterns and use the first that matches.

Subscale scores appear near their label text. Common patterns:

  Pattern A (most common — Workplace report):
    "Self-Regard\n...\n[NUMBER]"
    The score is typically an integer on its own line after the subscale name.

  Pattern B (Leadership report):
    "Self-Regard [NUMBER]"
    Score appears on the same line as the label.

  Pattern C (table format — some Group reports):
    Scores appear in a tab-separated table structure.

  Pattern D (360 reports):
    "Self   [NUMBER]    Observer  [NUMBER]"
    Two scores per line — self and observer average.

The parser must search the extracted text for EACH of the 15 subscale names
(accounting for minor variations in capitalization or hyphenation) and extract
the integer that appears nearest to that label within a defined character window.

---- SUBSCALE NAME VARIATIONS TO HANDLE ----

MHS uses slightly different label text across report versions. Handle all:

  "Self-Regard" / "Self Regard" / "SELF-REGARD"
  "Self-Actualization" / "Self Actualization" / "SELF-ACTUALIZATION"
  "Emotional Self-Awareness" / "Emotional Self Awareness" / "ESA"
  "Emotional Expression" / "EMOTIONAL EXPRESSION"
  "Assertiveness" / "ASSERTIVENESS"
  "Independence" / "INDEPENDENCE"
  "Interpersonal Relationships" / "Interpersonal" / "INTERPERSONAL REL."
  "Empathy" / "EMPATHY"
  "Social Responsibility" / "Social Resp." / "SOCIAL RESPONSIBILITY"
  "Problem Solving" / "Problem-Solving" / "PROBLEM SOLVING"
  "Reality Testing" / "Reality-Testing" / "REALITY TESTING"
  "Impulse Control" / "Impulse-Control" / "IMPULSE CONTROL"
  "Flexibility" / "FLEXIBILITY"
  "Stress Tolerance" / "Stress-Tolerance" / "STRESS TOLERANCE"
  "Optimism" / "OPTIMISM"
  "Total EI" / "Total Emotional Intelligence" / "Overall EI" / "EI Composite"
  "Well-Being" / "Well-Being Indicator" / "WBI" / "Happiness"

---- REPORT TYPE DETECTION ----

Detect report type from title text near the top of the first page:
  Contains "Workplace"   → WORKPLACE
  Contains "Leadership"  → LEADERSHIP
  Contains "Group"       → GROUP (always anonymized)
  Contains "360"         → THREESIXTY

  POST /api/eq/import/parse

    Accepts: multipart/form-data with field 'pdf' + JSON body 'options'

    Requires: auth middleware (coach role minimum)

  POST /api/eq/import/batch
    Import multiple PDFs in one operation.
    Accepts: multipart/form-data with fields 'pdfs[]' + JSON body 'batchOptions'
    Max files per batch: 20
    Returns: batch ImportResult summary
    Requires: auth middleware (coach role minimum)
    Note: Process files sequentially (not parallel) to avoid MongoDB
    connection pool exhaustion on Lambda.

  GET /api/eq/import/audit
    Returns paginated audit log for the authenticated coach's organization.
    Query params: page, limit, privacyMode, dateFrom, dateTo
    Returns: paginated list of audit log entries (never includes raw scores)

  GET /api/eq/import/status/:importId
    Returns status of a specific import (useful for batch polling).

  DELETE /api/eq/import/profile/:profileId
    Right-to-erasure endpoint. Deletes ALL data linked to a profileId:
    eq_profiles, eq_scores, eq_sessions, eq_reports, GROW plans.
    Does NOT delete the audit log entry (legal requirement).
    Does NOT delete norm contributions (already anonymized).
    Returns: confirmation with list of collections purged.

MONGODB CONNECTION — LAMBDA-SAFE PATTERN
================================================================================

If deploying to Lambda (App Runner or Beanstalk also fine), the MongoDB
connection MUST be cached outside the handler to prevent 4-5 second
reconnects on every invocation.

Create backend/src/config/db.ts with this exact pattern:

  import mongoose from 'mongoose';

  let cachedConnection: mongoose.Connection | null = null;

  export async function connectDB(): Promise<mongoose.Connection> {
    if (cachedConnection && cachedConnection.readyState === 1) {
      return cachedConnection;
    }
    const conn = await mongoose.connect(process.env.MONGODB_URI!, {
      serverSelectionTimeoutMS: 5000,
      maxPoolSize: 10,
      minPoolSize: 2,
    });
    cachedConnection = conn.connection;
    return cachedConnection;
  }

Call connectDB() at the TOP of every Lambda handler and every service method,
not inside route handlers. In Express, call it once at app startup.


================================================================================
VALIDATION RULES — ENFORCE IN BOTH BACKEND AND FRONTEND
================================================================================

Score validation (applied to every imported score):
  All 15 subscale scores: integer, inclusive range 55–145
  All 5 composite scores: integer, inclusive range 55–145
  Total EI: integer, inclusive range 55–145
  Well-Being Indicator: integer, inclusive range 55–145
  Composite plausibility: composite score should be within 20 points of the
    arithmetic mean of its 3 constituent subscales. Flag (do not block) if not.

Privacy mode validation:
  Mode A: consentObtained must be true — block import if false
  Mode A: clientName and clientEmail required
  Mode B: clientCode must follow format CLIENT-YYYY-NNN — validate with regex
  Mode B: offline mapping checkbox must be checked
  Mode C: no identifying fields accepted — strip silently if accidentally sent

File validation:
  PDF only — check MIME type (application/pdf) and magic bytes (%PDF-)
  Max file size: 10MB per file
  Max batch: 20 files
  Duplicate detection: SHA-256 hash of file content compared to existing
    pdfFilenameHash values in audit log — warn but do not block on duplicate

SECURITY REQUIREMENTS
================================================================================

  File upload:
    Validate MIME type server-side (not just client-side)
    Check PDF magic bytes: first 4 bytes must be 25 50 44 46 (%PDF)
    Scan for excessive file size before parsing
    Never store the uploaded PDF file — parse in memory, discard buffer
    Never log PDF content — only log metadata

  Data at rest:
    Mode A (identified): encrypt clientName, clientEmail, clientOrganization
    fields using AES-256-GCM before storing in MongoDB.
    Use a field-level encryption key stored in AWS Secrets Manager.
    All other fields (scores, metadata) stored unencrypted.

  Audit log:
    Never delete audit log entries — this is a legal compliance record
    The audit log stores field names, not field values
    pdfFilenameHash stored as SHA-256 — not reversible to original filename

  Access control:
    Only users with role 'coach' or 'admin' can access import endpoints
    Coaches can only import into their own organization (organizationId check)
    Audit log readable by coach and admin of same organization only

  Right to erasure:
    DELETE /api/eq/import/profile/:profileId removes:
